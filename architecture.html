Selendroid's Architecture

Overview about the selendroid-server

Selendroid is based on the Android instrumentation framework, so therefor only testing one app is supported. Selendroid-server in combination with the application under test (aut) must be installed on the device in order to be able to run automated end-to-end tests.

Selendroid Components

  1. Application under test (aut.apk)
  2. selendroid-server (selendroid-server.apk)
  3. webdriver client bindings from the selenium project
  4. your tests

The application under test must not be modified that means that the default version that is build for manual testing can be used. The only limitation is that this must be signed with the same Android debug certificate that was used to build selendroid-server (If you need to re-sign your APK I recommend to use the re-signer tool ).

Selendroid-server is implemented as an Android test project. The provided ruby gem contains a prebuild selendroid-server.apk and when running the command (after the gem was installed)

selendroid build yourApp.apk

a customized selendroid-server.apk will be build that has the target package of the specified app (yourApp.apk).
